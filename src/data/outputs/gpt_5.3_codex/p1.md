# クラウドとオンプレミスの違い（中学生向け）

## まず一言でいうと
- **クラウド**：インターネットの先にある「借りるコンピューター」
- **オンプレミス**：学校や会社の中に「自分で持つコンピューター」

たとえるなら、
- クラウドは「動画配信サービス（必要なときに使う）」
- オンプレミスは「自分の家にDVDやプレイヤーをそろえる」
の違いに近いです。

## 比較してみよう

| 観点 | クラウド | オンプレミス |
|---|---|---|
| サーバーの場所 | 事業者のデータセンター | 自分の建物・部屋の中 |
| 初期費用 | 小さめ（すぐ始めやすい） | 大きめ（機械を買う必要あり） |
| 月々の費用 | 使った分だけ増減しやすい | 保守費・電気代など固定費が多い |
| 使い始める速さ | 速い（数分〜数時間） | 遅め（準備に日数〜週単位） |
| 台数の増減 | かんたん（ボタン操作で増やせる） | 機械追加が必要で時間がかかる |
| 管理の手間 | 事業者が多くを担当 | 自分たちでほぼ全部担当 |
| ネット依存 | 高い（回線が必要） | 社内だけでも動かしやすい |

## それぞれの良い点・気をつける点

### クラウド
**良い点**
- すぐ始められる
- 必要な分だけ使える
- 災害対策やバックアップの仕組みを使いやすい

**気をつける点**
- 長時間・大量利用だと料金が高くなることがある
- インターネット障害の影響を受けやすい
- 設定ミスで情報が見えてしまうリスクがある（正しい設定が大事）

### オンプレミス
**良い点**
- 自分たちのルールで細かく管理しやすい
- 社外ネットワークに出さず運用しやすい
- 長期間の安定運用ではコスト予測がしやすい場合がある

**気をつける点**
- 最初に大きなお金が必要
- 故障対応・更新・セキュリティ対策を自分たちで行う必要がある
- 人手や専門知識がないと運用が大変

## どっちを選ぶ？
- **スピード重視・小さく始めたい** → クラウド向き
- **厳しい社内ルール・設備を自前で管理したい** → オンプレミス向き
- 実際は「大事なデータはオンプレ、一般サービスはクラウド」のように**両方を組み合わせる（ハイブリッド）**ことも多いです。

## まとめ
クラウドは「借りてすぐ使える便利さ」、オンプレミスは「自分で持って細かく管理できる安心感」が強みです。  
どちらが正解かは、**予算・スピード・安全性のルール・運用できる人材**で決まります。
