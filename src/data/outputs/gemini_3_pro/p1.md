# クラウドとオンプレミス：中学生でもわかる違い

コンピュータを使ってWebサイトやアプリなどのサービスを提供するには、「サーバー」という特別なコンピュータが必要です。このサーバーを**どこに置いて、誰が管理するか**の違いで、「クラウド」と「オンプレミス」の2つに分けられます。

わかりやすく「家」に例えてみましょう。

## 1. オンプレミス（On-Premises）＝「持ち家」

**オンプレミス**は、自分でサーバーを買ってきて、自分の会社の中に置いて管理する方法です。「自社運用」とも呼ばれます。

*   **イメージ:** 自分で土地を買って、家を建てて住む「持ち家」。
*   **特徴:**
    *   **自由:** 家の間取りや内装を好きに決められるように、システムを自分の好きなように細かくカスタマイズできます。
    *   **安全:** 自分の敷地内にあるので、情報の守り方を自分でコントロールできます。
    *   **大変:** 家の掃除や修理を自分でやるように、サーバーの故障対応やメンテナンスも全部自分たちでやらないといけません。
    *   **初期費用:** 家を建てるのにお金がかかるのと同じで、最初にサーバー機材を買うために高いお金がかかります。

## 2. クラウド（Cloud）＝「ホテル」や「賃貸」

**クラウド**は、インターネットを通じて、専門の会社（AmazonやGoogle、Microsoftなど）が持っているサーバーを借りて使う方法です。

*   **イメージ:** 必要な時だけ部屋を借りて泊まる「ホテル」や「賃貸マンション」。
*   **特徴:**
    *   **手軽:** ネットで申し込めば、すぐに使い始められます。面倒な機械の管理や修理は、貸してくれる会社がやってくれます。
    *   **柔軟:** 「急にお客さんが増えたから広い部屋にする」「いらなくなったから解約する」といった変更が簡単にできます。
    *   **始めやすい:** 最初に高い機械を買う必要がありません。使った分だけ料金を払う（電気代のような）仕組みが一般的です。
    *   **依存:** ネットが繋がらないと使えません。また、貸してくれる会社のルールに従う必要があります。

## まとめ：比較表

| 項目 | オンプレミス（持ち家タイプ） | クラウド（ホテル・賃貸タイプ） |
| :--- | :--- | :--- |
| **場所** | 自分の会社の中 | インターネットの向こう側 |
| **準備** | 機械を買って設置する（数週間〜数ヶ月） | クリック一つですぐ使える（数分） |
| **お金** | 最初にたくさん必要 | 使った分だけ毎月払う |
| **管理** | 自分たちでやる（大変） | 専門の会社にお任せ（楽） |
| **向いていること** | 独自の複雑なシステム、極めて高い機密情報の管理 | 新しいサービスの立ち上げ、アクセス数が変動するアプリ |

昔は「オンプレミス」が当たり前でしたが、今は手軽でスピードが速い「クラウド」を使うことが非常に増えています。
