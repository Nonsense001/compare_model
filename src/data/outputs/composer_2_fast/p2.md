# API通信を料理に例えると

**API（Application Programming Interface）**は、アプリ同士が「お願い」と「お返事」をやり取りするための**決まりごと**です。これを料理にたとえると次のようになります。

## 厨房とホール

- **お客様（アプリ）**が、メニュー表を見て注文する  
  → これが **リクエスト**（「この料理ください」という依頼）。

- **厨房（サーバー側）**がレシピに従って調理し、皿に盛って出す  
  → これが **処理**。

- **給仕が料理を運ぶ**  
  → これが **レスポンス**（「はい、ご注文の〇〇です」という結果の返却）。

## メニュー表がAPIの仕様

どんな料理を頼めるか、値段はいくらか、アレルギー表記はあるか——**メニューに書いてある約束**が、プログラムでは **APIドキュメント** に相当します。書いていない料理は頼めませんし、書き方が違うと厨房が混乱します（＝パラメータの形式が合わないとエラー）。

## まとめ

API通信は、「決められた注文のしかたで厨房に伝え、決められた形で料理が返ってくる」流れです。料理に例えると、**注文のルール・調理・配膳**がセットになったものだとイメージすると分かりやすいです。
